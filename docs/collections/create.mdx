## Creating a collection

To create a collection, you need to pass the collection name and the schema definition to the `create()` method on the`client.collections()` instance.

<CodeGroup>

```python Python
from topk_sdk.schema import int, text, vector, vector_index, keyword_index, semantic_index

client.collections().create(
    "books",
    schema={
        "title": text().required().index(keyword_index()),
        "title_embedding": f32Vector(dimension=1536)
            .required()
            .index(vector_index(metric="euclidean")),
        "published_year": int().required(),
    },
)
```


```typescript Javascript
import { int, text, vector, vectorIndex, keywordIndex, semanticIndex } from "topk-js/schema";

await client.collections().create("books", {
  title: text().required().index(keywordIndex()),
  title_embedding: f32Vector({ dimension: 1536 })
    .required()
    .index(vectorIndex({ metric: "euclidean" })),
  published_year: int().required(),
});
```

</CodeGroup>

## Schema

## Data types

TopK supports the following data types. There are more on our roadmap and we are working effortlessly to bring them to you as soon as possible.

### int()

`int()` function is used to define an integer field in the collection schema.

<CodeGroup>

```python Python
from topk_sdk.schema import int

"published_year": int()
```


```typescript JavaScript
import { int } from "topk-js/schema";

"published_year": int()
```

</CodeGroup>

### float()

`float()` function is used to define a float field in the collection schema.

<CodeGroup>

```python Python
from topk_sdk.schema import float

"price": float()
```


```js JavaScript
import { float } from "topk-js/schema";

"price": float()
```

</CodeGroup>

### bool()

`bool()` function is used to define a boolean field in the collection schema.

<CodeGroup>

```python Python
from topk_sdk.schema import bool

"is_published": bool()
```


```typescript JavaScript
import { bool } from "topk-js/schema";

"is_published": bool()
```

</CodeGroup>

### text()

`text()` function is used to define a text field in the collection schema.

<CodeGroup>

```python Python
from topk_sdk.schema import text

"title": text()
```


```js JavaScript
import { text } from "topk-js/schema";

"title": text()
```

</CodeGroup>

### f32_vector()

`f32_vector()` function is used to define a vector field with 32-bit floating point values. This function takes `dimension` as a parameter (greater than 0).


<CodeGroup>

```python Python
from topk_sdk.schema import f32_vector

"title_embedding": f32_vector(dimension=1536)
```


```js JavaScript
import { f32Vector } from "topk-js/schema";

"title_embedding": f32Vector({ dimension: 1536 })
```

</CodeGroup>

<Note>
  The vector dimension will be validated when upserting documents.
  Passing a vector with a different dimension will result in an error.
</Note>

### u8_vector()

`u8_vector()` function is used to define a vector field with `u8` values. You can pass vector `dimension` as a parameter (required, greater than 0) which will be validated when upserting documents.

<CodeGroup>

```python Python
from topk_sdk.schema import u8_vector

"title_embedding": u8_vector(dimension=1536)
```


```js JavaScript
import { u8Vector } from "topk-js/schema";

"title_embedding": u8Vector({ dimension: 1536 })
```

</CodeGroup>

### binary_vector()

`binary_vector()` function is used to define a binary vector packed into `u8` values. You can pass vector `dimension` as a parameter (required, greater than 0) which will be validated when upserting documents.

<Note>
  Binary vector dimension is defined in terms of the number of bytes. This means that for a 1024-bit binary vector, the dimension `topk` expects is **128** (1024 / 8).
</Note>

<CodeGroup>

```python Python
from topk_sdk.schema import binary_vector

"title_embedding": binary_vector(dimension=128)
```


```js JavaScript
import { binaryVector } from "topk-js/schema";

"title_embedding": binaryVector({ dimension: 128 })
```

</CodeGroup>

### bytes()

`bytes()` is used to define a bytes field in the schema.

<CodeGroup>

```python Python
from topk_sdk.schema import bytes

"image": bytes()
```


```js JavaScript
import { bytes } from "topk-js/schema";

"image": bytes()
```

</CodeGroup>

## Properties

### required()

`required()` is used to mark a field as required. All fields are `optional` by default.

<CodeGroup>

```python Python
"title": text().required()
```


```js JavaScript
"title": text().required()
```

</CodeGroup>

The above example shows how to mark a field `title` as required.

## Methods

### index()

`index()` is used to create an index on a field.

#### `semantic_index()`

This method is used to create both a vector and a keyword index on a given field. This allows you to do both semantic search and keyword search over the same field. Note that `semantic_index()` can only be called over `text()` data type.

<CodeGroup>

```python Python
from topk_sdk.schema import semantic_index

"title": text().index(semantic_index())
```


```typescript JavaScript
import { semanticIndex } from "topk-js/schema";

"title": text().index(semanticIndex())
```

</CodeGroup>

Optionally, you can pass a `model` parameter to the `semantic_index()` method. Supported models are:

- `cohere/embed-english-v3`
- `cohere/embed-multilingual-v3` (default)

You can also pass an optional `embedding_type` parameter to the `semantic_index()` method as well. TopK supports the following embedding types for Cohere models:

- `float32`
- `uint8`
- `binary`



#### `vector_index()`

This method is used to create a vector index. Only fields with `f32_vector()`, `u8_vector()`, or `binary_vector()` data types
can be indexed with a vector index.

<CodeGroup>

```python Python
from topk_sdk.schema import vector_index

"title_embedding": vector(dimension=1536).index(vector_index(metric="cosine"))
```


```js JavaScript
import { vectorIndex } from "topk-js/schema";

"title_embedding": f32Vector({ dimension: 1536 }).index(
  vectorIndex({ metric: "cosine" })
),
```

</CodeGroup>

The above example shows how to create a vector index on a field `title_embedding` with the `cosine` similarity metric.
However, there are more metrics available:

- `euclidean`
- `cosine`
- `dot_product`
- `hamming` (only supported for `binary_vector()` type)

#### `keyword_index()`

This method is used to create a keyword index on a `text()` field.
`index(keyword_index())` can only be called over `text()` data type.

<CodeGroup>

```python Python
from topk_sdk.schema import keyword_index

"title": text().index(keyword_index())
```


```js JavaScript
import { keywordIndex } from "topk-js/schema";

"title": text().index(keywordIndex())
```

</CodeGroup>
